1. Постановка задачи:
Реализовать сервис коротких ссылок.

2. Требования:
1) Веб-сервер поддерживает одновременно несколько соединений.
2) Для каждого соединения по HTTP-запросу вида "POST полная_ссылка" сервер генерирует, записывает в БД и возвращает короткую ссылку.
3) Для каждого соединения по HTTP-запросу вида "GET короткая_ссылка" сервер находит в БД полную ссылку и делает на нее редирект; если полная ссылка не найдена, сервер возвращает ошибку 404.
4) Время хранения коротких ссылок ограничено и задается отдельным конфигурационным параметром.
5) Конфигурация сервера задается в виде JSON-файла и загружается при старте сервера.
6) Необходим набор тестов для системного тестирования.
7) Необходимо документировать код и приложение (файл README.md).

3. Условия:
1) Сервер состоит из отдельных модулей, отвечающих за работу с БД, обслуживание соединений и т.д.
2) Конфигурация сервера задается в виде JSON-файла, путь к которому передается серверу при запуске через ключ -c.
3) В приложение предусмотрен логгер с настраиваемыми уровнями логирования.
4) Используемые функции и алгоритмы снабжены информативными комментариями.
5) Тесты оформлены в отдельном каталоге/файле.
6) Код соответствует PEP8.
